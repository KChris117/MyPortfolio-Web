BAT 3D Printer Request App

An internal web application for managing and tracking 3D printing requests in a manufacturing environment.

The Story Behind 3D Printer App

The use of 3D printing technology for prototyping and specialized components has increased significantly in the industry. However, the process of submitting 3D printing requests was previously handled manually through verbal communication or messaging apps, leading to miscommunication, unorganized queues, and difficulties in tracking job statuses.

To resolve these pain points, the BAT 3D Printer Request Web App was created. This centralized platform digitalizes the entire submission process, allowing users to submit detailed requests while providing administrators with a powerful dashboard to manage the production queue (Pending, On Progress, Completed, Rejected) in real-time.

This project successfully brought transparency, structure, and efficient documentation to the 3D printing request lifecycle.

Key Features

  • Request Management: Digital forms for users to submit 3D printing requirements (filament type, urgency level, site area, and design attachments).
  • Admin Dashboard: A centralized control panel for administrators to manage requests, update statuses, and add completion notes.
  • Real-time Tracking: Users can effortlessly track their request history and monitor the status of their ongoing jobs.
  • Role-Based Access Control: Secure login and authorization managed by ASP.NET Identity to distinguish between Users and Admins.
  • Data Privacy: Ensures the confidentiality and security of uploaded design files.

Tech Stack

Framework: ASP.NET MVC (C#)

Database: SQL Server & Entity Framework

Frontend: Razor View Engine, HTML, CSS, JavaScript, Bootstrap

Architecture: Model-View-Controller (MVC) & Waterfall Methodology
